4) Electronics Required (V1 BOM) + Notes/Warnings/Alerts
4.1 Core Compute + UI
Required

Raspberry Pi 4 (4GB is OK for V1) (you already have this)

7-inch touchscreen (you already have this)

Strongly recommended add-ons

Storage: A2-rated microSD or USB 3.0 SSD (preferred for stability and fast DB reads)

Cooling: heatsink + fan (or a good passive heatsink case)

Alerts

ALERT: Thermal throttling will cause UI lag and slow queries; cooling is not optional for “instrument feel”.

ALERT: Cheap microSD cards cause random DB slowness and corruption risk.

4.2 Power System (Portable “Instrument” Build)
Required (portable mode)

2S Li-ion pack (7.4V nominal) + 2S BMS (protection)

5V / 5A buck converter (stable power to Pi + screen)

Power switch + soft shutdown button (recommended)

Recommended

USB-C PD trigger (9V/12V) for flexible charging input

Battery fuel gauge module (accurate battery %)

RTC module (timestamp integrity in reports) 

helixia pro

Warnings

WARNING: Power brownouts will corrupt the DB pack and reports. Use a stable 5V/5A supply and proper wiring.

WARNING: No hard power-off while writing reports / updating DB packs. Always soft shutdown.

4.3 Security / Integrity (Optional but “Patent-Strength” feel)
Optional (recommended for V1+)

Secure element for device-signed reports 

helixia pro

Notes

Not needed to make V1 work, but it upgrades trust: “instrument-generated outputs” rather than “app output.”

4.4 Suggested “Safe Build” Wiring Notes

Use proper gauge wiring for 5A lines (don’t run Pi + screen through thin jumper wires).

Add inline fuse on battery output.

Keep buck converter output stable at 5.1V under load (Pi 4 is picky).
