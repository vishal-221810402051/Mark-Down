# AI Data Processing System — Technical Specification

This document describes the architecture and implementation roadmap for a small AI data processing system built around Raspberry Pi hardware.

---

## Project Overview

The goal of the system is to ingest environmental sensor data, process it locally using machine-learning models, and generate predictions for anomaly detection.

Key capabilities:

- Local data ingestion
- Lightweight ML inference
- Hardware monitoring
- Dashboard visualization

---

## Hardware Platform

| Component | Qty | Purpose |
|---|---|---|
| Raspberry Pi 5 | 1 | Main processing unit |
| NVMe SSD | 1 | Data storage |
| DS18B20 Sensor | 1 | Temperature monitoring |
| Camera Module | 1 | Image capture |

## Setup Guide

To initialize the environment run the following commands.

npm install
npm run build
npm run lint
docker compose up --build -d

Verify the service:

curl http://localhost:3001/health

## Database Schema

CREATE TABLE sensor_data (
  id INTEGER PRIMARY KEY,
  sensor_id TEXT,
  temperature REAL,
  created_at TIMESTAMP
);

CREATE TABLE predictions (
  id INTEGER PRIMARY KEY,
  reading_id INTEGER,
  label TEXT,
  confidence REAL
);

## Monitoring Workflow

Steps:

1. Start the system
2. Verify sensor readings
3. Run prediction pipeline
