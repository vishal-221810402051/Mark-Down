M.E.D.I.S.
Modular Emergency Drug Intelligent Synthesizer
Autonomous Portable Medicine Formulation Platform

Inventor & Lead Systems Architect: Vishal Nelaturi
Patent: French Patent Application FR2513603
Current Maturity: TRL 3 – Research Platform

1. Executive Summary

M.E.D.I.S. (Modular Emergency Drug Intelligent Synthesizer) is a modular, electromechanical pharmaceutical formulation platform designed to produce liquid medications on demand through automated ingredient handling, controlled dissolution, microfluidic mixing, and inline quality validation.

The system integrates:

Precision volumetric dosing

Controlled fluid routing

Microfluidic formulation

Inline spectroscopy and sensor validation

Embedded control architecture

The objective is to replace rigid pre-manufactured medicine logistics with a flexible on-demand formulation platform capable of operating in remote or infrastructure-limited environments.

Potential future deployment scenarios include:

Remote medical clinics

Disaster relief operations

Military field medicine

Research stations

Long-duration space missions

At the current stage, M.E.D.I.S. is developed as a research-stage formulation platform, not a clinical medical device. The focus is on subsystem validation and architecture de-risking before regulatory escalation.

2. Problem Context

Modern medicine supply chains rely on pre-manufactured pharmaceutical products such as tablets, capsules, syrups, and injectables.

This approach presents several limitations:

Large logistics footprint

Medicine expiration and wastage

Lack of dosage flexibility

Incompatibility with remote environments

Dependency on centralized pharmaceutical manufacturing

Existing automated pharmaceutical systems are large laboratory compounding robots, requiring stable infrastructure, trained operators, and controlled environments. They are not portable, modular, or autonomous.

There is currently no widely deployed system capable of:

Portable medicine formulation

Modular chemical ingredient handling

Integrated dosing intelligence

Inline quality validation

3. Core System Concept

M.E.D.I.S. is designed as a modular electromechanical formulation platform composed of specialized subsystems responsible for ingredient handling, formulation, validation, and control.

The system architecture integrates:

Solid and liquid ingredient handling

Dissolution and hydration processing

Precision volumetric dispensing

Microfluidic formulation

Spectroscopic validation

Automated cleaning and sterilization cycles

The system structure emphasizes modular isolation of functional domains, enabling independent subsystem validation and reducing integration risk.

4. System Architecture Overview

The platform is composed of six primary modules.

Module 1 — Gravity-Fed Cartridge Bank

The cartridge bank stores active pharmaceutical ingredients (APIs), buffers, excipients, and system fluids.

Cartridges are vertically oriented to enable gravity-assisted feed into downstream modules.

Cartridge categories include:

API cartridges

Buffer/excipient cartridges

Cleaning fluid cartridges

Eccentric vibration motors prevent powder bridging and ensure controlled material flow.

External reservoirs provide:

Sterile water

Phosphate buffers

Citrate buffers

Ethanol-water mixtures

Acid/base solutions

Quality-control reagents

Internal reservoirs manage:

Intermediate formulations

Cleaning solvents

Flush solutions

Waste collection

These reservoirs are routed through a central valve manifold that directs fluids across system modules.

Module 2 — API Hydration & Pre-Dissolution

Solid ingredients must be dissolved before controlled formulation.

This module performs dissolution using multiple agitation mechanisms:

Magnetic stirring

Ultrasonic agitation

Mechanical vibration

Controlled heating (25–60°C)

Sensors monitor:

Temperature

Turbidity

Fluid levels

Inline filtration removes particulates before fluid transfer.

This module ensures consistent solution preparation prior to dosing operations.

Module 3 — Precision Syringe Dispensing Core

The dosing engine is based on a multi-syringe volumetric dispensing platform.

Current configuration:

Seven syringe channels

Servo-controlled plungers

Lead-screw linear actuation

Stepper-driven plate translation

This design allows microliter-to-milliliter dosing precision.

Two operational architectures exist:

Mode 1 — Single-Stepper Synchronized System

All syringes operate together through a shared linear translation mechanism.

Mode 2 — Multi-Stepper Group Architecture (Future)

Syringes are divided into functional groups:

APIs

Buffers/excipients

Cleaning agents

Independent stepper motors enable parallel dispensing and pathway isolation.

This architecture improves contamination control and throughput.

Module 4 — Microfluidic Mixing Network

Formulation occurs inside PDMS microfluidic chips bonded to glass substrates.

The microfluidic network includes:

Serpentine mixing channels

Metered dosing outlets

Micro-valves

Quality-control chambers

Laminar flow within microchannels enables predictable mixing with minimal dead volume.

Microfluidic control ensures precise formulation reproducibility.

Module 5 — Quality Control & Spectroscopy

The QC module validates formulation quality before dispensing.

Sensors include:

Visible spectrum spectroscopic sensors

RGB colorimetric sensors

Turbidity sensors

Pressure sensors

Flow sensors

Temperature probes

Optical inspection camera

Quality validation methods include:

Spectral pattern analysis

Optical inspection

Concentration comparison

Flow stability checks

Machine learning models classify sensor readings and detect anomalies.

If validation fails, the system triggers automatic abort and cleaning routines.

Module 6 — Control, Power & Sterilization

The control architecture includes:

Raspberry Pi orchestration layer

Microcontroller-based actuator control

Vision inspection subsystem

Sensor acquisition modules

Sterilization routines include:

Chemical flushing

Microfluidic cleaning cycles

Acid/base cleaning protocols

Optional UV-C sterilization

These mechanisms maintain fluid pathway integrity between runs.

5. Embedded Intelligence Architecture

M.E.D.I.S. integrates multiple computational engines:

Dosage Engine

Determines safe formulation parameters using rule-based constraints and machine learning models.

Quality Validation Engine

Classifies spectroscopic and optical results using AI-assisted pattern recognition.

Process Control Engine

Coordinates actuator motion, microfluidic routing, and safety interlocks.

These engines ensure automated supervision of formulation processes.

6. System Safety Architecture

The platform incorporates several safety layers.

Automatic shutdown occurs if:

QC validation fails

Pressure anomalies occur

Sensor readings deviate from expected ranges

Dosing exceeds safety thresholds

Failure modes currently analyzed include:

overdosing

underdosing

microfluidic clogging

sensor failure

cross contamination

power interruption

Detection mechanisms include step-count verification, spectroscopic validation, flow monitoring, and watchdog timers.

8. Regulatory Development Strategy

Regulatory alignment follows a staged escalation approach.

Stage 1 — Technical Feasibility (Current)

Subsystem validation and fluid handling architecture testing.

Stage 2 — Risk Mapping & Documentation

Hazard analysis aligned conceptually with ISO 14971.

Stage 3 — Regulatory Classification

Future analysis under EU MDR 2017/745 if intended for clinical deployment.

Potential classification hypothesis:

Class IIb medical device (if integrated into a medical workflow).

10. Strategic Vision

M.E.D.I.S. aims to pioneer a new paradigm in distributed pharmaceutical production.

Instead of transporting finished medicines globally, the system enables localized medicine synthesis from modular ingredients.

The long-term vision includes:

decentralized pharmaceutical infrastructure

adaptive formulation capabilities

medicine production in extreme environments

Future development may support clinical integration, regulated pharmaceutical workflows, and autonomous medical supply systems.
